2. ✓ Редактор кода обраний та встановлений;

Я вирішив встановити Visual Studio Code та додатково Sublime Text, щоб спробувати. Його беззаперечним плюсом було те, що він має Portable-версію і не вимагає інстацяції.

2. ✓ Надрукувати базовий "скелет" html-файлу;

Мушу визнати, що не зовсім уявляю, що саме "запхати" в той базовий скелет файлу і які теги та стилі обов'язково мають бути використані, щоб робота відповідала вимогам. Тому спробував створити те, що Ви бачите на зараз.

Sublime Text, до речі, відразу після спроби написати "<html>" самотужки створив велику частину стандатрного скелету, але нам ці подачки не треба! Тому я почав спочатку в Visual Studio Code. Та і певні атрибути тегу мета, які були створенні автоматично, я не розумів.

Цікавий факт, який дізнався, поки писав попередній параграф:
Не можна просто так взяти і написати "<html>" в коді сторінки та сподіватися, що браузер цей тег покаже.

Маю питання по лекції

На даному зобреженні елементи "<title>", "<script>", "<h1>","<h2>" позначені, як сестринські елементи.
Має питання таке: чи дійсно всі 4 ці елементи сестринські? Адже хоч вони і мають спільного предка "<html>", все ж батьківські елементи для них різні. І сестринськими будуть окремо "<title>", "<script>" та окремо "<h1>","<h2>" і"<p>".

Ще одне питання виникло буквально щойно. Мій Visual Studio Code написав, що в мене забагато Юнікод-символів і перестав їх виділяти. Очевидно, я щось роблю не так. Можете підказати, що саме?
